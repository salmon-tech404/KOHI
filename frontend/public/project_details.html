06
Thông báo đơn hàng
Sắp tới
Gmail (Nodemailer)
Gửi email xác nhận đơn hàng cho khách sau khi đặt thành công
Dùng Nodemailer + Gmail App Password — miễn phí
Email template HTML — tóm tắt đơn, tên khách, giờ lấy hàng
Gửi thông báo cho admin khi có đơn mới
Zalo Notification (qua SĐT)
Zalo OA (Official Account) — tạo tài khoản doanh nghiệp
Gửi tin nhắn Zalo qua số điện thoại khách hàng — cần đăng ký Zalo OA, có thể mất phí theo gói
Zalo API — gửi ZNS (Zalo Notification Service) template message
ZNS tính phí theo tin nhắn (~50-200đ/tin). Cần đăng ký template được Zalo duyệt trước.
Khuyến nghị: Bắt đầu với Gmail (miễn phí, dễ tích hợp). Zalo ZNS phù hợp khi có lượng khách ổn định vì có chi phí vận hành — nên cân nhắc sau khi go-live.
07
Security & Production
Sắp tới
API Security
Rate limiting — giới hạn số request từ 1 IP (express-rate-limit)
Helmet.js — bảo vệ HTTP headers
Input validation — sanitize dữ liệu từ form (express-validator)
CORS — chỉ cho phép domain frontend được phép gọi API
Auth Security
bcrypt — hash password admin trước khi lưu
JWT expiry + refresh token — token hết hạn tự động
httpOnly cookie — tránh XSS đánh cắp token từ JS
Production Deploy
Backend deploy lên Railway hoặc Render
Frontend deploy lên Vercel hoặc Netlify
Domain + HTTPS (Cloudflare hoặc nhà cung cấp VN)
Environment variables — tách biệt .env dev và production
Lưu ý: Security làm sau cùng khi tính năng đã ổn định. Ưu tiên: Rate limit → CORS → Helmet → bcrypt → JWT httpOnly. Deploy sau khi security đã pass review.
