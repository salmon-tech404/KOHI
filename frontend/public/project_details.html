01
Foundation & Homepage
Hoàn thành
Setup
Khởi tạo project (React + Vite + Tailwind + Express + MongoDB)
Cấu hình Tailwind v4 — @theme custom colors & fonts
Design token system — @layer components (section-label, btn-*, nav-*, card-*, item-*, footer-*)
Kết nối MongoDB Atlas
Cấu trúc thư mục feature-based (layouts / sections / ui / store / constants)
constants/images.js + constants/menu.js — single source of truth
Homepage UI
Navbar — transparent → blur on scroll, cart icon
Hero — fullscreen ảnh, animation fade-in, 2 CTA buttons
WhyUs — 3 điểm nổi bật với số trang trí
MenuPreview — grid 4 sản phẩm, hover reveal button
OriginsSection — 3 vùng trồng cà phê, hover reveal description
BeansSection — Arabica / Robusta, editorial layout, object-contain image
StorySection — background image vườn cafe, dark overlay
Footer — giờ mở cửa, địa chỉ, liên hệ
Reusable Components
PageHero.jsx — layout component dùng chung cho các hero section
MotionCard.jsx — wrapper component tái sử dụng animation
ProductCard.jsx — hover effect, AnimatePresence text swap
PropTypes cho toàn bộ components có props
02
Giỏ hàng & Đặt hàng
Đang làm
Cart
cartStore.js — Zustand store (items, addItem, removeItem, updateQty, clearCart, getTotalPrice, getTotalItems)
CartDrawer.jsx — panel slide-in từ phải, overlay backdrop, tăng/giảm số lượng
Navbar — hiển thị badge số lượng, click mở CartDrawer
ProductCard — "Thêm vào giỏ" gọi addItem() từ store
Checkout

POST /api/orders — gửi đơn lên backend
Trang OrderSuccess — xác nhận đơn hàng + thông tin

Product model + routes (GET /api/products)

api/orders.js — hàm gọi API tạo đơn hàng

Thay data tĩnh (constants/menu.js) bằng API call thật

05
Admin Dashboard
Sắp tới
Quản lý đơn hàng
Trang /admin — xem danh sách đơn hàng hôm nay
Cập nhật trạng thái: pending → confirmed → ready → done
Quản lý sản phẩm
CRUD sản phẩm — thêm, sửa, xóa, ẩn/hiện
Upload ảnh sản phẩm
Quản lý nội dung (Mini CMS)
ContentBlock model — lưu nội dung Spotlight, Banner trong MongoDB
Form chỉnh sửa text, font size, ảnh cho từng section
Thay vì sửa code, quản lý vào admin page chỉnh trực tiếp
Blog posts — thêm/sửa/xóa bài viết
Frontend render nội dung dynamic từ API thay vì hardcode
Lưu ý: Admin dashboard không cần JWT phức tạp. Có thể dùng simple password hoặc IP whitelist để bảo vệ route /admin.
